Intro

My goal is to find an organization that meets these standards and is able to utilize my tech-oriented skillset to help meet their goals. Projects

Pantree

For my senior capstone project, I worked on a team of four to develop a kitchen management app called Pantree that aims to reduce food waste by keeping inventory of the food in your pantry and suggesting recipes that use food items closest to expiration. We used Google Firebase Firestore as our backend database along with the Flutter and Dart technologies to inevitably deploy the project across multiple platforms including Android, iOS, and the web.

I mainly contributed to the pantry and shopping list views along with the app's overall front-end design. The features I worked on for both views include the pantry/list selector drop-down, the ability to add/remove pantry/list items, the ability to edit and remove a pantry/list, and the item cards for each food item. In addition to these features, I also helped to keep our codebase organized, fixed bugs when they arose, and improved the style of the UI across the app.

Local P2P File Transfer System

One of my most enjoyed projects in the CS program was creating a local peer-to-peer (P2P) file transfer system for my Computer Networks class. This simple P2P network was written in Java and works by running 3 separate servers and nodes (clients). The user must specify the ports for each server to run on as a single command-line argument. When the user runs a node, they will be prompted to specify those 3 server ports along with the ID for the node. The node IDs must be either 1, 2, or 3, and they must be distinct. After the user inputs all the required data, each node will connect to a "home" server first (each node's home server will be different) along with the other two servers. The server is multithreaded, allowing for multiple clients to connect at once, however, the client code only runs on a single thread. Once the clients are all connected to each other, the user waits until the same line is displayed in every terminal before continuing. The auto-setup system will elect a leader node and disconnect the non-leader peers from each other. From there, the non-leaders will specify the names of the files they want to send, taking caution to not proceed before both nodes specify the files they want to share. The non-leaders will then specify the names of the files they want to retrieve. The retrieved files will be located in the "ReceivedFiles" directory. Once the user is finished, they can type "exit" on each peer to end the session.
